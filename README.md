# Calendar Feed — фид событий групп VRChat для мира 5ul

Раз в 30 минут GitHub Actions логинится бот-аккаунтом в VRChat API, забирает
события календарей групп из `groups.json` и публикует компактный
`docs/events.json` на GitHub Pages. Мир читает его через VRCStringDownloader —
домен `*.github.io` входит в доверенный список String Loading, так что
игрокам НЕ нужно включать «Allow Untrusted URLs».

## Установка (один раз, ~15 минут)

1. **Бот-аккаунт VRChat.** Создай отдельный аккаунт (не основной!). Включи в
   его настройках 2FA типа **Authenticator App (TOTP)** и сохрани base32-секрет
   (строка вида `ABCD EFGH ...`, показывается при настройке). Вступи этим
   аккаунтом во все группы из списка — иначе он увидит только публичные события.

2. **Репозиторий GitHub.** Создай **публичный** репозиторий (Pages на
   приватном — платная фича), залей в него содержимое этой папки.

3. **Секреты.** Settings → Secrets and variables → Actions → New repository
   secret, четыре штуки:
   | Секрет | Значение |
   |---|---|
   | `VRC_USERNAME` | логин бот-аккаунта |
   | `VRC_PASSWORD` | пароль бот-аккаунта |
   | `VRC_TOTP_SECRET` | base32-секрет TOTP |
   | `VRC_CONTACT` | твой e-mail (обязателен в User-Agent по правилам VRChat API) |

4. **GitHub Pages.** Settings → Pages → Source: `Deploy from a branch`,
   Branch: `main`, папка `/docs`. Через минуту фид будет доступен по адресу
   `https://<логин>.github.io/<репозиторий>/events.json`.

5. **Группы.** Впиши в `groups.json` реальные ID групп (`grp_...` — виден в
   URL страницы группы на vrchat.com). Поле `name` — только для логов,
   отображаемые имена и цвета настраиваются в Unity-окне календаря.

6. **Первый запуск.** Actions → update-calendar-feed → Run workflow. Проверь,
   что ран зелёный и `docs/events.json` появился. Дальше — само по расписанию.

7. **Unity.** В Unity: Tools → Calendar → Настроить календарь, вставь URL фида
   из шага 4, добавь те же группы, нажми «Собрать панель».

## Как это устроено

- Кука сессии хранится в `state/cookies.json` и переживает запуски через
  actions/cache: повторные логины по паролю ограничены лимитом сессий VRChat,
  поэтому логин происходит только когда кука умерла.
- Скрипт ходит в API с паузами 3 с и бэкоффом на 429 — правила VRChat для
  неофициального API (внятный User-Agent + щадящий темп).
- События обрезаются: горизонт 35 дней, описание до 280 символов, символы вне
  запечённого TMP-атласа мира заменяются пробелами.
- Если события не изменились, файл не переписывается (heartbeat `gen`
  обновляется раз в 6 часов) — история коммитов не засоряется.

## Известные риски

- **API неофициальный.** VRChat терпит аккуратные интеграции, но эндпоинты
  могут поменяться без предупреждения. Красный ран в Actions = сходить сюда
  и посмотреть лог.
- **Cloudflare против IP датацентров.** Если раны стабильно падают с 403 на
  логине — GitHub-раннеры попали в блок. Варианты: self-hosted runner дома,
  запуск скрипта по расписанию на своей машине (Планировщик задач Windows),
  или запасной источник vrcwiki.ru/api/events.
- **Сессия привязана к IP** (исторически). Раннеры меняют IP каждый ран, кука
  может протухать чаще обычного — скрипт тогда просто перелогинится.
